## 使用案例

| 使用案例 | 個人檔案作業 |
| ----- | ----- |
| 使用案例描述 | 人員可以預覽會員的個人檔案，會員可以觀看其他會員的公開個人檔案、自己的個人檔案可選擇不公開，預設公開、可以修改個人檔案、可以看到自己的貼文、粉絲、追蹤者，系統管理人可以觀看所有人的個人檔案  |
| 主要參與者 | 人員、會員、系統管理人  |
| 利害關係人與目標 | 人員:能預覽會員的個人檔案  |
|   | 會員:可觀看其他會員的公開個人檔案、可選擇不公開檔案、可修改個人檔案、可看到自己的貼文、粉絲、追蹤者 |
|   | 利害關係人:可觀看所有人的個人檔案 |
| 前置條件 | 會員、系統管理人須先登入  |
| 後置條件 | 選擇不公開檔案後，系統會將公開轉成私人  |
| 主要成功情節 | 1.人員點擊會員帳號  |
|   | 2.可預覽該會員的個人檔案 |
|   | 3.會員登入後，點擊個人檔案 |
|   | 4.點擊後可以看到自己的貼文、粉絲、追蹤者 |
|   | 5.點擊編輯個人檔案可修改個人檔案 |
|   | 6.點擊隱私設定和帳號安全可選擇不公開帳號 |
|   | 7.點擊其他會員帳號後可觀看該會員的公開個人檔案 |
|   | 8.系統管理人登入後可看到所有會員帳號 |
|   | 9.點擊會員帳號後可以看到該會員的個人資料 |
| 例外情節 | 6a.若不選擇不公開帳號，則個人檔案繼續公開  |
|   | 7a.若該會員有設定不公開個人檔案，則系統顯示此個人檔案不公開 |
| 其他需求 | 點擊粉絲後除了可以看到追蹤的用戶之外也可以看到系統推薦的用戶  |

| 後置條件 | 可以在系統上出現搜尋紀錄  |

|   | 2.點擊搜尋 |
|   | 3.搜尋想找的會員 |
|   | 4.點擊該會員可以查看該會員 |
|   | 5.返回主畫面再次點擊搜尋會出現之前的搜尋紀錄 |
